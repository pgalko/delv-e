# --- scipy API changes ---
scipy.stats.theilslopes returns (slope, intercept, low_slope, high_slope), NOT .lower/.upper
pymannkendall is not installed — use scipy.stats.kendalltau(x, y) for Mann-Kendall test
scipy.stats.pearsonr returns (statistic, pvalue) — access as r, p = pearsonr(x, y), not result.statistic

# --- pandas deprecations ---
DataFrame.append() is removed — use pd.concat([df1, df2]) instead
Use df.groupby(col, observed=True) for categorical columns to avoid FutureWarning

# --- common statistical mistakes ---
scipy.stats.levene and scipy.stats.bartlett require array-like inputs, not DataFrames — use .values
When using scipy.stats.f_oneway, drop NaN first — it does not handle NaN gracefully
mannwhitneyu alternative defaults to 'two-sided' in recent scipy — specify explicitly if needed

# --- numpy/type traps ---
np.float64 values in f-strings print as "np.float64(3.14)" — cast with float() for clean output
When comparing pandas Timestamps, use pd.Timestamp() not datetime.datetime()

# --- financial / time-series libraries ---
prophet: use `from prophet import Prophet`, not `from fbprophet import Prophet` (package renamed)
arch: GARCH fitting uses `arch_model(returns, vol='Garch', p=1, q=1)` — pass returns not prices
hmmlearn: GaussianHMM requires 2D input — reshape with `.reshape(-1, 1)` for single-feature series
ta: indicators expect lowercase columns (high, low, close, volume) — rename or pass columns explicitly
ruptures: use `rpt.Pelt(model="rbf").fit(signal)` — signal must be a numpy array, not a Series
pmdarima: use `import pmdarima as pm; model = pm.auto_arima(series)` — not `from pyramid.arima`

# --- data cleaning traps ---
Dollar-prefixed columns (e.g. "$352.82") are strings — strip '$' and cast: df[col].str.replace('$','').astype(float)
Date columns in M/D/YYYY format: use pd.to_datetime(df['Date'], format='%m/%d/%Y') — do NOT assume YYYY-MM-DD
Column names with slashes (e.g. 'Close/Last') — use df['Close/Last'] not df.Close/Last (dot access fails)

# --- matplotlib chart quality ---
Synthesis charts: do NOT add text annotations, arrows, or floating statistics — the report text provides all numbers
Keep charts clean: only title, axis labels, tick labels, and legend. No ax.annotate, no ax.text callouts

# --- computational limits (execution is killed after 300s) ---
Bootstrap/permutation/cross-validation: use ≤200 iterations, ≤5 CV folds. Do NOT use 1000+ iterations.
Iterative model fits (MixedLM, Bradley-Terry, EM, MCMC): filter to groups with ≥10 observations BEFORE fitting, cap at ≤300 groups.
Pairwise operations (distance matrices, all-pairs tests, correlation matrices on entities): cap at ≤500 entities per dimension, or sample.
Large regression/OLS: if >10,000 rows, aggregate to per-group summaries before fitting iterative solvers.
Grid search (GridSearchCV, parameter sweeps): limit to ≤50 total parameter combinations.
When stratifying models across eras/groups: fit sequentially and store only summary statistics. Clear intermediate results between strata.
Avoid nested loops that multiply: N_iterations × N_strata × N_entities must stay under 500,000 total operations.
